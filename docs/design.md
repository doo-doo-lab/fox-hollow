# 小狐大世界 - 项目策划文档

> 最后更新：2026-05-02

## 一、项目概述

- **项目名称**：小狐大世界（Fox Hollow Tales）
- **类型**：增量放置 / 村庄建设
- **灵感来源**：Kittens Game（猫国建设者），玩法机制相似，但所有文案、命名、主题完全原创
- **主题**：玩家经营一个狐狸山谷村落，从采集野莓开始，逐步建造设施、吸引村民、发展科技
- **风格**：白底黑字纯文字界面，紧凑单行排版
- **线上地址**：https://2d9lpj7e.mule.page/

#### 资源发展梯度

资源体系遵循严格的**从低到高**发展梯度，参照猫国建设者的层层递进模式（猫薄荷→木→矿→铁→煤→钢→钛→合金→铀→反物质）：

```
第一纪：野莓 / 圆木 / 碎石 / 兽皮
    ↓ 加工
第二纪：木板 / 砖块 / 矿铁 / 铜钱 / 卷轴 / 学识
    ↓ 加工
第三纪：彩络 / 醴浆 / 墨锭 / 香草 / 丝帛 / 古币
    ↓ 分支
第五纪·工业：煤 → 钢 → 火油 → 寒钛 → 辉石 → 重晶
第五纪·灵修：灵能 → 命丝 → 共振子 → 晶丝 → 元念
```

**硬约束**：高阶建筑/配方的造价以同阶或上一阶资源为主。基础资源仅可作为微量辅料，不可作为高阶内容的主要消耗。详见 `RULES.md` §1.4。

### 2.7 季节系统

每年4个季节循环，每季 100 天（游戏内时间）。

| 季节 | 野莓产量倍率 | 特殊效果 |
|------|-------------|----------|
| 早春 | ×1.5 | 丰收季 |
| 盛夏 | ×1.0 | 正常 |
| 金秋 | ×1.2 | 略丰 |
| 寒冬 | ×0.25（庇护后×0.4） | 产量骤降，需提前囤粮 |

### 2.8 满意度系统

- 基础值：100%
- 人口 > 5 时，每多1只狐狸 -2%
- 灵狐祠每座 +5%
- 研究「狐灵传说」+10%
- 范围：10% ~ 200%
- 影响手动采集野莓的产量
- 影响全体职业产出（乘以满意度百分比）
- 悬浮面板显示计算明细及当前职业产出倍率

### 2.9 人口系统

- 狐狸通过「狐狸窝」提供容量（每座+2）
- 当容量有余且野莓 > 20 时，有 1.5% 概率每 tick 迁入新狐狸
- 研究「先祖之眼」后狐狸消耗野莓 -15%（0.35→0.30/s）
- 野莓耗尽时，每 50 tick 有一只狐狸离开
- 每只狐狸消耗野莓 0.35/s

### 2.10 事件系统

#### 山谷见闻
- 25 种随机事件，约每 10 分钟触发一次（概率 0.0003/tick）
- 四大类：灵启（需灵狐祠）、天象、山谷异象、过客与声音
- 加权随机选取，部分有季节/建筑前置条件
- 部分事件附带资源奖励

#### 世界的回响
- 18 段极稀有叙事文本，约每 3 小时触发一次（概率 0.00002/tick）
- 按季节分类（春4/夏5/秋5/冬4），仅当前季节可触发
- 斜体蓝灰色显示，与普通日志区分
- 纯叙事，无资源效果

### 2.11 存档系统

- **自动存档**：localStorage，每 30 秒自动保存
- **手动存档**：Header「存档」按钮
- **存档码**：Base64 编码导出/导入，支持跨设备迁移
- **后台补算**：切换标签页再切回时，自动补算离开期间的资源产出（含季节更替、狐狸消耗、饥荒、迁入）
- 最大补算时间 24 小时

## 三、UI 设计

- **风格**：白底黑字，纯文本，无图标无装饰
- **布局**：三栏（左：资源 | 中：操作 | 右：日志）
- **建筑行**：单行紧凑「名称 (数量) 费用 [建造] [出售]」
- **费用不足**：红色高亮
- **Tab**：营火 / 村落 / 工坊 / 研究

### 3.1 悬浮信息面板

所有游戏元素名称支持鼠标悬停显示信息面板（`position: fixed`，JS 定位防溢出）：
- 建筑：描述 + 效果数值 + 解锁提示 + 短句
- 职业：描述 + 每人产出 + 短句
- 研究：描述 + 加成百分比 + 短句
- 工坊：描述 + 短句
- 资源：短句 + 特殊提示
- 满意度：显示计算明细

### 3.2 短句系统

- 数据中以 `tip` 数组存储，支持多句
- 每季缓存一次，季节更替时刷新
- 风格：冷幽默、有哲思、克制、口语化

**斥候显示**：斥候无 `*P` 产出键，`jobEffects()` 对斥候返回特殊文本"出征远行队伍来源；授业可提升远行奖励"。JD 定义中 `e: {}`（空对象），另加 `desc` 字段供 UI 读取。授业按钮 hover 文案对斥候独立成"消耗 X 卷轴，所有远行奖励倍率 +10%"。

**斥候出征流程**（v0.13.0 改动）：
- `canSendExp`：检查 `G.job.scout?.c > 0`（不再用 freeFox）
- `sendExpedition`：扣 `G.job.scout.c -= foxCount`，`G.foxAway += foxCount`
- `resolveExpedition`：归队 `G.foxAway -= foxCount`，`G.job.scout.c += foxCount`
- 由于 freeFox 重算公式 `foxes - foxAway - sum(jobs)` 中 `sum(jobs)` 减少与 `foxAway` 增加抵消，闲置狐狸数不变

### 8.7 远行系统

#### 操作流程

1. 在「山外」Tab 选择目的地
2. 消耗资源（野莓干粮 + 铜钱路费 + 符咒灵力）
3. 派出 1-3 只斥候（从 `G.job.scout.c` 中扣除，锁定为外出状态）
4. 等待游戏内天数倒计时
5. 返回：获得资源奖励 + 触发叙事（占位，待审核）

#### 远行记录结构

`G.expeditions` 数组，每条记录：

```
{
  dest: string,          // 目的地 ID（nearHill / forest / oldRuin / cloudRidge）
  foxCount: number,      // 派出狐狸数
  ticksLeft: number,     // 剩余 tick 数
  totalTicks: number,    // 总 tick 数（用于进度条计算）
  usedSpiritPath: boolean // 是否已对此次远行施放过灵路
}
```

#### 目的地

| 目的地 | 路程 | 消耗 | 解锁条件 |
|--------|------|------|----------|
| 旧墟 | 30天 | 野莓50, 铜钱2 | 驿道×1 |
| 云岭 | 60天 | 野莓100, 铜钱5, 符咒1 | 驿道×1, 瞭望塔×1, 研究：远途跋涉 |
| 荒丘 | 100天 | 野莓200, 铜钱8, 符咒4 | 驿道×3, 瞭望塔×1 |
| 密林 | 150天 | 野莓300, 铜钱10, 符咒8 | 驿道×3, 灵狐祠×2 |

路程受瞭望塔缩减：每座 ×0.85（2 座 = ×0.72，3 座 = ×0.61）。

#### 奖励池

每次返回从目的地奖励池抽取 2-3 项，数量 = 基础数量 × scoutBonus × (研究「远途跋涉」? 1.5 : 1)，其中 `scoutBonus = 1 + 在岗斥候 × 0.05 + 斥候授业次数 × 0.10`（v0.13.0 起）。

| 目的地 | 奖励池（基础数量） |
|--------|-------------------|
| 旧墟 | 矿铁 3-6, 古币 1-2, 遗光 1（10%概率），叙事碎片（必定） |
| 云岭 | 符咒 3-5, 古币 2-3, 叙事碎片（必定） |
| 荒丘 | 碎石 10-20, 兽皮 1-3, 香草 1（40%概率） |
| 密林 | 圆木 15-25, 兽皮 2-4, 丝帛 1（40%概率） |

旧墟和云岭每次必定返回一段叙事碎片（按顺序推进，不重复抽取）。

#### 抉择事件

在岗斥候 ≥ 1 + 30% 概率（v0.13.3 起；旧版 ≥ 2）时，远行返回有概率触发抉择事件。玩家从 2 个选项中选择，不同选项导致不同后果（资源变化、新事件加入事件池、特殊状态等）。无最优解。

抉择事件内容全部为占位符，待审核后填入。

#### 叙事碎片收集状态

已收集的叙事碎片以 `G.narratives` 持久化：

```
G.narratives = {
  oldRuin: [],      // 已解锁的旧墟碎片索引数组，如 [0, 1, 2]
  cloudRidge: []    // 已解锁的云岭碎片索引数组
}
```

远行返回时，取 `G.narratives[dest].length` 作为下一条碎片的索引，若未超出数据总条数则推入。山外 Tab"叙事碎片"区域按序展示已收集条目，未收集的显示为"???"占位。

#### 外出狐狸状态

- 新增 `G.foxAway` 追踪外出狐狸数
- 派出时（v0.13.0 起）：`G.foxAway += n`，`G.job.scout.c -= n`（不再扣 freeFox）
- 返回时：`G.foxAway -= n`，`G.job.scout.c += n`，并主动重算 freeFox
- 由于 `freeFox = foxes - foxAway - sum(jobs)`，sum(jobs) 与 foxAway 同步增减，freeFox 数值在派遣前后不变
- 外出狐狸**不消耗村内野莓**（出发时已扣干粮）
- 外出狐狸**不受饥荒减员影响**
- 野莓消耗计算用 `G.foxes - G.foxAway`
- `rmFox()` 只从村内狐狸（含职业和闲置）中移除

### 8.8 商队系统

外来商队为其他物种的狐族或山谷中的灵兽，周期性到访。

- 每 2-3 个季节随机有商队到访，停留 1 个季节后离开
- 到访时日志提示（占位），「山外」Tab 显示商队商品

- 冷却追踪：`G.caravanTimer`（整数），每次季节更替 +1；商队离开时重置为 0；当 `caravanTimer >= 2` 时每季有 50% 概率到访（保证间隔 2-3 季）；商队在场时不计数

| 白鹤信使 | 卷轴×2 = 铜钱12 | 符咒×2 = 铜钱10 | 砖块3 → 铜钱5 | 灵狐祠×1 |
| 旧墟遗民 | 古币×2 = 铜钱15 | 符咒×3 = 铜钱12 | 不收购 | 远行旧墟完成≥1次 |

## 九、图纸与专精系统（v0.12.0，设计中）

### 9.1 设计理念

增量游戏中后期最大的问题是"堆数量无决策"——建筑造到 10 座和造到 20 座之间只有数值差异，没有策略差异。图纸与专精系统通过**不可撤销的方向选择**为每座建筑和每种职业引入分岔路径，让不同玩家的村庄产生实质性的结构差异。

获取方式绑定商队的随机到访和概率携带，保持放置游戏"等待+抓机会"的节奏——图纸来了就看看要不要买，不来就继续挂机。

### 9.2 图纸

图纸是一种**具名物品**，不是通用资源。每张图纸对应一个建筑或职业的某个专精方向。

- 存储：`G.blueprints` 数组，元素为 `{ id: 'berryPatch_A', target: 'berryPatch', spec: 'A' }`
- 图纸不可堆叠，同一种图纸只能持有一张（已拥有则商队不会再带）
- 同一目标的 A/B 图纸互斥持有：若已持有 berryPatch_A，则 berryPatch_B 不会出现在商队中，反之亦然
- 已激活的专精对应图纸从背包移除；若激活 A 时背包中持有同目标的 B，B 自动丢弃
- 在「工坊」Tab 新增「图纸」区域展示已持有的图纸，点击可查看效果并激活

### 9.3 商队携带图纸

商队到访时有 **50% 概率**额外携带一张图纸。图纸种类从该商队的图纸池中随机抽取（排除玩家已拥有、已激活、以及同目标另一方向已持有的）。**图纸仅在玩家拥有对应资源上限时才会生成**（例如香草上限 > 0 才会出现需要香草的图纸）。

| 商队 | 图纸池 |
|------|--------|
| 山猫行商 | 莓果园 / 伐木场 / 采石坑专精、采集者天赋 |
| 河獭商队 | 鞣革坊 / 锻造炉 / 储藏窖专精、伐木工 / 矿工天赋 |
| 白鹤信使 | 藏书阁 / 灵狐祠专精、学者 / 铁匠天赋 |
| 旧墟遗民 | 猎手 / 商贩天赋 |

图纸价格 = 铜钱底价 + 该商队对应的稀有资源。**v0.13.5 起按强度分两档**（S 强档 / A 普档），同商队两套价：

| 商队 | S 档（强力图纸） | A 档（其他） |
|------|-----------------|--------------|
| 山猫行商 | 铜钱 180 + 香草 1 | 铜钱 100 + 香草 1 |
| 河獭商队 | 铜钱 225 + 丝帛 1 | 铜钱 125 + 丝帛 1 |
| 白鹤信使 | 铜钱 270 + 古币 1 | 铜钱 150 + 古币 1 |

**强档（10 张）的判定标准**：纯强力、无副作用 / 不带消耗惩罚 / 不带条件限制：
- 建筑专精 4 张：松鼠病（上限翻倍）/ 四眼田狐（学识 +50% + 卷轴 +30%）/ 生狐勿近（学识上限翻倍）/ 一颗矿掰两颗用（全建筑铁造价 -15%）
- 职业天赋 6 张：跟树杠上了（+40% 木）/ 只挖一个坑（+40% 石）/ 憋着（+50% 兽皮）/ 发呆冠军（+40% 学识 + 30% 卷轴）/ 叮叮当当（+50% 铁）/ 算盘精（+50% 铜钱）

UI 显示：S 档图纸名字用深金黄 `#b8860b`（`.bp-item-name-s`），A 档用默认棕色 `#9a7b4f`，玩家一眼能看出强弱。

### 9.6 引擎实现要点

#### 专精效果生效

`calcR()` 中在建筑产出和职业产出阶段检查 `G.bldSpec[id]` / `G.jobTalent[id]`，应用对应的乘数或附加产出。专精效果以**乘法叠加**应用在基础产出上，不与研究加成冲突。

示例：莓果园「沃土」激活后，`berryP` 产出从 `0.15/s × 座数` 变为 `0.15 × 1.4 × 座数`。

#### 造价修正

锻造炉「一颗矿掰两颗用」、铁匠「边角料」等造价减免效果通过新函数 `specCostMul(id, resKey)` 返回造价乘数，`bp()` 中调用。多个造价修正以**乘法叠加**，如「一颗矿掰两颗用」×0.85 × 丰收祭 ×0.7 = ×0.595。

#### 消耗类专精

鞣革坊「厚韧」的"全村野莓消耗 +10%"仅作用于 `foxEatRate()` 返回的基础狐狸食量，**不**放大采集者「勤爪」的额外 0.05/s 消耗。两者各自以独立加项叠加在 `r.berry` 上：先扣 foxEat × 1.1 × 村内狐狸数，再扣 0.05 × 采集者数。

伐木场「深林」每座消耗碎石 0.005/s，在 `calcR()` 中作为负产出叠加：
```
if (G.bldSpec.lumberYard === 'A') r.stone -= 0.005 * G.bld.lumberYard.c * 5;
// ×5 因为 tick 间隔 200ms，0.005/s = 0.001/tick × 5
```

#### 薄削连续转化

鞣革坊「薄削」以连续速率建模，在 `calcR()` 中叠加负兽皮产出和正铜钱产出（仅当兽皮 > 0 且铜钱未满时生效）：
```
if (G.bldSpec.tannery === 'B' && G.res.leather.v > 0 && G.res.coin.v < G.res.coin.mx) {
  r.leather -= 0.01 * 5;   // -0.01/s
  r.coin += 0.005 * 5;     // +0.005/s
}
```

#### 灵术消耗折扣

灵狐祠「引灵」的灵术消耗 -20% 修改 `canSpell()` / `castSpell()` 中的费用检查与扣除逻辑。折扣后费用向上取整（`Math.ceil`），与商队造价折扣保持一致：

```
var mul = G.bldSpec.shrine === 'B' ? 0.8 : 1;
for (const p of SD[id].cost)
  if (G.res[p.r].v < Math.ceil(p.a * mul)) return false;
```

示例：祈雨符咒消耗 5 × 0.8 = 4.0 → `Math.ceil(4.0)` = 4；灵狐庇护符咒消耗 8 × 0.8 = 6.4 → `Math.ceil(6.4)` = 7。

#### 上限翻倍类专精交互

储藏窖「深窖」与藏书阁「秘阁」均含"翻倍"效果，需明确作用范围：

- **深窖**：将储藏窖自身在 `calcMx()` 中提供的各资源上限加成翻倍（即 `e[k+'Mx'] * s.c` 变为 `e[k+'Mx'] * s.c * 2`），**仅影响储藏窖自身的上限贡献**。
- **秘阁**：将藏书阁自身的 `loreMx` 贡献翻倍（同理仅乘于藏书阁的 `loreMx * s.c`），**不**翻倍其他建筑（如储藏窖）对学识上限的贡献。

两者同时激活时，储藏窖的 `loreMx` 贡献 ×2，藏书阁的 `loreMx` 贡献 ×2，总学识上限 = 基础 + 储藏窖贡献×2 + 藏书阁贡献×2 + 其他建筑贡献，**不是** 4 倍。

#### 研究费用折扣

学者「划重点」的研究费用 -10% 需新增 `researchCostMul()` 函数，返回当前研究费用乘数。同时修改 `canU()` 和 `research()` 中的费用检查与扣除逻辑，应用该乘数。

#### 商队图纸生成

`trySpawnCaravan()` 成功生成商队后，50% 概率（商贩「野路子」激活时为 60%）调用 `rollBlueprint(caravanId)`，从该商队图纸池中随机抽一张。排除规则：
- 已在 `G.blueprints` 中持有的
- 已在 `G.bldSpec` / `G.jobTalent` 中激活的
- 同目标另一方向已在 `G.blueprints` 中持有的
- 对应资源上限为 0 的（玩家无法支付）

池中无可选项时不生成。结果存入 `G.caravan.blueprint`：
```
G.caravan = { id: picked, bought: {}, blueprint: { id, target, spec, cost } | null }
```

购买图纸后，执行 `G.caravan.bought['blueprint'] = true`（与普通商品购买一致），将图纸写入 `G.blueprints`，并产生日志："购入图纸：[图纸名]。"

激活专精/天赋时同样产生日志："激活专精：[建筑名]「[专精名]」。" / "激活天赋：[职业名]「[天赋名]」。"

## 十、文化经济循环（v0.15.0 - v0.15.1，已完成）

### 10.1 设计理念

v0.14 引入彩络/醴浆/墨锭三种文化品后，它们缺乏持续消耗场景（仅用于一次性风俗激活），同时香草/丝帛获取路径过少。v0.15 通过节令系统、文化灵术、建筑维持三个消耗端 + 多路径获取扩展，形成三环嵌套的资源循环：

- **内环（基础循环）**：野莓/圆木/碎石/兽皮 → 加工品 → 建筑/研究
- **中环（贸易循环）**：香草/丝帛 ↔ 彩络/醴浆/墨锭 → 节令消耗 + 灵术消耗 + 建筑维持
- **外环（灵性循环）**：符咒 → 灵术 → 临时增益 → 推动内环/中环效率

### 10.2 节令系统

研究「百艺通觉」完成后启用，每季手动选择消耗文化品换取季节加成。

**双模式**（`G.riteMode`）：
- `'auto'`（默认）：季节切换时按上次选择静默应用；玩家可在风俗 Tab 顶部 banner 展开调整
- `'manual'`：季节切换时弹出选择面板，必须确认或跳过

| 档位 | 消耗 | 效果 |
|------|------|------|
| 彩络 | 彩络 ×1 | 全职业产出 +5% |
| 醴浆 | 醴浆 ×1 | 野莓产量 +8% |
| 墨锭 | 墨锭 ×1 | 学识产出 +10% |
| 三全礼 | 三者均选 | 额外满意度 +5% + 全产出 +3% |

- 资源不足自动跳过该项
- 离线跨季按 `G.lastSeasonRites` 静默应用，归来一次性 toast 总结
- 首次解锁全屏引导面板（`G.riteIntroSeen` 持久化）

### 10.3 文化灵术（3 个）

覆盖现有灵术未触及的维度：资源上限、工坊产出、研究费用。

| 灵术 | 消耗 | 效果 | 限制 |
|------|------|------|------|
| 盈库 | 彩络 ×2 | 本季资源上限 +30%，满仓溢出半效累积 | 每季一次 |
| 双工 | 醴浆 ×2 | 本季工坊产出 +50%（加法叠加 craftMul） | 每季一次 |
| 墨契 | 墨锭 ×2 | 下次研究 -40% + 下支商队图纸概率 +15%，限本季 | 用掉/季末前不可重复 |

### 10.4 香草获取扩展（5 条路径）

| 路径 | 机制 | 期望年产量 |
|------|------|-----------|
| 猎手「林间采风」 | 季末概率判定 `min(0.5, 0.12×猎手数 + 0.03×授业等级)` | ~1.6（3 猎手授业 1） |
| 商贩「路边生意」 | `G.merchantSpiceAcc` 累积 ≥15 时 35% 概率触发 | ~0.6（3 商贩商驿 1） |
| 商队到访赠礼 | 商队生成时 40% 概率附赠 1 香草 | ~0.8 |
| 旧墟远行掉落 | 25% 概率获 1 香草（v0.14.2 加入） | ~0.5 |
| 山猫行商购买 | 2 香草 / 8 铜钱（现有） | ~1.0 |

### 10.5 丝帛获取扩展（3 条路径）

| 路径 | 机制 | 解锁条件 |
|------|------|----------|
| 纺丝帛 | 兽皮 5 + 圆木 5 → 丝帛 1（工坊配方） | 艺工坊 ×1 |
| 换丝帛 | 香草 3 + 铜钱 5 → 丝帛 1（工坊配方） | 商驿 ×1 |
| 河獭商队 | 2 丝帛 / 10 铜钱（现有） | — |

远途跋涉研究丝帛费用从 2 降至 1（v0.14.2）。

### 10.6 建筑维持消耗

参考猫国"建筑需要养"模式，半效运作（不归零）。

| 建筑 | 维持 | 正常效果 | 断供效果 |
|------|------|----------|----------|
| 月歌台 | 每年春季扣 1 醴浆/座 | 符咒 +0.003/s/座 | 符咒产出 ×0.25 |
| 艺工坊 | 每季扣 1 彩络/座 | craftCultureMul +0.05/座 | 加成 ×0.5 |

### 10.7 商队/图纸概率元加成

| 加成维度 | 来源 | 数值 | 上限 |
|----------|------|------|------|
| 商队来访概率 | 共聚堂 ×座数 | +2%/座 | +10% |
| 商队携带图纸概率 | 藏书阁 ×座数 | +2%/座 | +10% |
| 商队携带图纸概率 | 墨契灵术（一次性） | +15% | — |
| 商队携带图纸概率 | 商贩「野路子」天赋（现有） | 50% → 60% | — |

总概率上限 95%。

### 10.8 文化研究双分支（v0.14.2）

树荫堂之后形成「歌与祭」/「艺与聚」双路线：

  │                └─ 连枝溯本 → 祖龛

### 11.1 已实现（v0.16.0 第一批）

v0.16.0 实现了基于可切换政体的初版政治系统：

- **议事录**：累积型知识资源，无上限，年初+20，议事堂被动+0.0005/s/座
- **议事堂**：木板15+砖块10+圆木50+铜钱20，产出议事录+学识加成
- **政堂**：木板25+砖块20+矿铁10+铜钱30，政体正面效果+5%/座（上限5座）

- **6 政体**：长老议会/公议会/商道议会/谷无主/隐居宗/武德同，可变更（代价300议事录+100古币+1季满意度-20%）
- **4 政策域**：土地/教育/通商/阶层，各3选项，首选免费切换有冷却

### 11.2 重构方向（roadmap-v2 §四）

v0.16 正在从"可切换政体"向"分层不可逆路线树"重构：

**核心变更**：
1. 取消议事录专用资源 → 花费走通用资源（卷轴+铜钱+古币）
2. **谷声方案已废弃**（roadmap-v2 §四 1.1 决议移除）：用研究链门控（councilLore → polityLore → policyLore）+ 通用资源花费替代

4. 政策从可切换 → 不可逆
5. 政体从 6 选 1 可切换 → Tier 1（2选1）→ Tier 2（6选1，受级联）→ Tier 3（政策域，受级联锁定）
