#### 阶段 0 审查修复记录

1. **[bug] apply/restore 禁用建筑 LIFO 顺序**：`applyFogDisabled()` + `applyDemonDisabled()` 按 FIFO 叠加减少建筑计数，但 restore 也按 FIFO 会导致同时受两套禁用的建筑计数恢复错误。修复：restore 改为 LIFO 顺序（先 `restoreDemonDisabled()` 再 `restoreFogDisabled()`），在 `tick()` 和 `simulateOffline()` 中同步修正。
2. **[防御] 燃料门控交叉排除**：`calcEnergy()` 原本排除 `energyP` / `pollutionP`，新增排除 `leylineP` / `unrestP`；`calcLeyline()` 原本排除 `leylineP` / `unrestP`，新增排除 `energyP` / `pollutionP`。防止未来跨系统建筑的系统字段被误判为燃料消耗。
3. **[bug] bldEffects() 通用 `*P` 处理器优先级**：原代码中通用 `k.endsWith('P')` 分支位于所有显式处理器之前，导致 `energyP` / `pollutionP` 等显式处理器成为死代码（被通用分支短路）。修复：将通用 `*P` 兜底移至 if/else if 链末尾，确保 `energyP` / `pollutionP` / `leylineP` / `unrestP` 走各自的专用格式化。此为工业线遗留 bug，在灵修线审查时一并修复。
4. **[补全] resBreakdown() 资源分解面板**：新增灵脉衰减显示（3 处：建筑被动、专精副产、进阶副产），与 energyRatio 对称；新增躁念惩罚对符咒/学识产出的分解提示。
5. **[补全] 资源 tooltip 躁念惩罚**：在污染惩罚行后新增躁念惩罚行（符咒 charmM、学识 loreM）。

## 三、灵脉系统（类比工业能量）

灵脉不是资源，是**产消平衡值**。每秒灵脉产出 - 每秒灵脉消耗 = 净灵脉。

- 净灵脉 ≥ 0：一切正常
- 净灵脉 < 0：所有耗灵建筑效果按缺口比例衰减（`ratio = prod / cons`，效果 ×ratio）

### 灵脉管理博弈

- 前期只有聚灵阵（+1/座），需要堆数量
- 中期共振塔提供高灵脉（+3），但本身消耗灵能——灵能不能全拿去做配方
- 后期寂石窟是灵脉主力（+5），但消耗晶丝——晶丝也是C阶段核心建材
- **D阶段灵脉危机**：镜灵台(-3)、元念炉(-4)、寂界门(-6)、灵契祭坛(-2) 共增加 -15 灵脉消耗，需要大量寂石窟(+5)来平衡
- **E阶段灵脉终极**：渊核熔炉(-8)、星幕穹顶(-10) 是最大灵脉消耗者；星幕穹顶的"负灵脉时50%保底"是安全网设计
- 与工业能量的区别：工业靠化石燃料（油），灵修靠高阶资源反哺（灵能、晶丝）

## 十一、灵契系统（D 阶段，对标猫国契约）

> 猫国的契约系统通过黑金字塔数量乘以契约效果，持续产出稀有资源。灵修对应为灵契祭坛×灵契。

### 核心机制

- **解锁条件**：灵契学研究 + 灵契祭坛×1
- **灵契祭坛作用**：每座祭坛是灵契产出的"倍率底座"，灵契产出 = 基础值 × 祭坛数量 × (1 + 升级加成)
- **灵契上限**：同时只能激活 1 种灵契（通过"灵契祈愿"灵术切换）
- **所有灵契都有持续资源消耗**——灵契不是免费被动，是一种"以某种资源换另一种资源"的长期投资

### 灵契列表（5 种）

| 灵契 | 效果（/tick/祭坛） | 持续消耗（/tick/祭坛） | 解锁条件 | 对标猫国 |
|------|-------------------|----------------------|---------|---------|
| 命纹契 | 命丝 +0.02 | 灵能 0.05 | 灵契学 | 独角兽契约 |
| 镜魂契 | 镜灵碎片 +0.005（积满10碎片→1镜灵） | 共振子 0.08 | 灵契学 + 镜灵台×1 | 天角兽契约 |
| 寂渊契 | 寂石 +0.01 | 晶丝 0.03 + 躁念额外+0.02/s | 灵契学 + 寂界门×1 | 死灵兽契约 |
| 幽典契 | 幽典碎片 +0.003（积满10→1幽典） | 元念 0.06 | 灵契学 + 幽典阁×1 | 眼泪契约 |
| 永弦契 | 永弦碎片 +0.001（积满20→1永弦） | 寂石 0.05 | 灵契学 + 寂界门×1 + 永弦术研究 | 时间水晶契约 |

### 灵契博弈

- **命纹契**：低门槛，用灵能换命丝。前中期最实用，后期价值递减。
- **镜魂契**：镜灵是D阶段稀缺资源，此契约是镜灵的唯一被动来源（配方是主动的），但共振子消耗大。
- **寂渊契**：寂石产出好，但会额外增加躁念——是唯一产生躁念的灵契。高风险高回报。
- **幽典契**：幽典是终局材料前置，此契约让幽典不再只靠手动配方。元念消耗意味着和元念池争资源。
- **永弦契**：极慢但极珍贵。永弦是E阶段核心资源，提前在D阶段缓慢积累。寂石消耗意味着和寂界门/寂石冥想争资源。

### 与猫国契约的设计差异

| 维度 | 猫国契约 | 灵修灵契 |
|------|---------|---------|
| 同时激活 | 可同时激活多个 | 同时只能1个（通过灵契祈愿切换） |
| 倍率建筑 | 黑金字塔（单一建筑） | 灵契祭坛（D阶段专用建筑） |
| 消耗模式 | 无消耗，纯被动 | 持续消耗不同资源（需主动管理） |
| 切换成本 | 无 | 灵契祈愿灵术（消耗灵核+幽典） |

## 十二、寂石冥想机制详述（对标猫国燃烧时间水晶）

> 猫国的"燃烧时间水晶"是后期核心循环之一：消耗时间水晶跳过若干年，加速积累所有产出。灵修的寂石冥想是相同定位。

### 基础规则

- **触发**：施放"寂石冥想"灵术（D阶段灵术）
- **前置**：寂界门×1
- **每次消耗**：寂石×5 + 元念×10（基础）
- **跳跃天数**：10 + 5×寂界门数量（天）
- **结算**：跳跃期间所有建筑/职业的产出按跳跃天数一次性结算（含灵契产出）
- **躁念**：跳跃期间躁念正常按速率累积（惩罚生效）
- **冷却**：900s（15分钟）

### 升级增强

| 升级 | 效果 | 前置 |
|------|------|------|
| 深寂冥想 #115 | 冥想跳跃天数 +50% | 寂界术 + 寂界门×2 |
| 寂石冥想精修 #116 | 冥想消耗 -30% | 寂弦感应 + 寂界门×3 |
| 元念冥想 #117 | 冥想期间元念产出额外 ×2 | 元念锻冶 + 元念炉×2 |
| 长寂入定 #118 | 冥想冷却 -40%（900s→540s） | 深层灵脉 + 寂界门×3 |

### 核心博弈

- 寂石冥想消耗**寂石**——寂石也是永弦（E阶段核心）的前置材料
- 在D阶段大量冥想加速会延缓E阶段准备；在E阶段冥想则加速终局材料积累
- 躁念在冥想期间正常累积：如果当前躁念控制不好，冥想会让惩罚恶化
- 与灵契的资源竞争：寂渊契消耗晶丝产出寂石，寂石又被冥想消耗，形成循环

### 与猫国时间水晶的设计差异

| 维度 | 猫国燃烧时间水晶 | 灵修寂石冥想 |
|------|----------------|-------------|
| 前置建筑 | 超时空传送仪 | 寂界门 |
| 消耗资源 | 时间水晶（1个/次） | 寂石×5 + 元念×10 |
| 跳跃时间 | 固定（受升级影响） | 可变（随寂界门数量增长） |
| 副作用 | 时间悖论（概率丢资源） | 躁念正常累积（确定性惩罚） |
| 产出结算 | 全产出一次性获得 | 全产出一次性获得（含灵契） |
| 冷却 | 无冷却（有资源就能烧） | 900s冷却（可通过升级缩短） |

### 核心差异（灵修独特性）

1. **灵术驱动 vs 建筑驱动**：工业靠建建筑解决问题，灵修靠施放灵术解决问题（灵术有消耗、有冷却、需要主动操作）
2. **循环感 vs 线性感**：工业是不可逆的科技树推进，灵修灵术是季度循环的节奏感
3. **自动化方式不同**：工业通过研究永久解锁自动制作，灵修通过灵术"灵织"临时启动（需每季施放，可通过升级延长）
4. **进阶体现方式**：无显性境界数字，玩家通过"能建什么建筑、能施什么灵术、能跑什么任务"来感知自己的阶段——就像猫国玩家通过"能探索什么星球"来感知进度

## 十八、终局接口预留（副线 hook point）

> 灵修主线设计灵迁结局。+神启→改终局为神升。通达**不改变终局**，仅在结局画面增加通达专属叙事段落。本节声明主线中哪些节点是副线可以覆写的 hook point，避免后续设计需要回改主线文档。

### 覆写规则

1. **纯加法**：副线的覆写应尽量是"追加"而非"替换"。例如 H1 是追加前置条件，H6 是追加新灵契，H5 是追加新升级。
2. **终局由神启决定**：H3/H4 的终局灵术/任务名仅在神启激活时改变（灵迁→神升）。通达不改变终局，只追加叙事段落。
3. **不修改 A-D 内容**：副线 hook 全部在 D 末期和 E 阶段。A-C 阶段内容不受副线影响。
4. **资源命名空间**：副线引入的新资源（盟约露等）在各自副线文档中定义，不出现在本文档（神铃已删除，见勘误E30）。
