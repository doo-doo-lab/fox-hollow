# 秘仪路详细设计（灵修+神启副线）

> 副线定位：灵修主线之上的信仰叠加层。信仰形态 = 个人化密教修行走向飞升。
> 对标：密教模拟器（Cultist Simulator）飞升路线 × 古希腊厄琉息斯秘仪 × 奥尔弗斯教。
> 一句话：**你不是在修炼灵术，你是在一步步变成神。**
> 4 阶段递进，体量约为主线的 70%。
> 8 资源，22 建筑，15 配方，6 职业，35 研究，100 升级。

> ⚠️ **mulerun 接手开发须知**：本文档是**详细设计文档**，不是开发执行手册。
> 接手开发请先按顺序读：[INDEX.md](INDEX.md) → [roadmap-v2.md §四 实施进度表](roadmap-v2.md) → [DEV_SOP.md](DEV_SOP.md) → [RULES.md §八](../RULES.md)。
> 不要直接复制本文档代码到 data.js。秘仪路实施在 Phase 4（v0.19）才开始，当前先聚焦 Phase 1。

### 开发规则

1. **不写短句（tip）**：除非明确要求，实现步骤时不主动编写 tip 短句/风味文字。
2. **遇到问题或抉择先查猫国百科**：设计不确定时优先参考 `reference/` 目录下的猫国建设者百科。
3. **兼容游戏逻辑与资源体系**：所有新增内容使用 `sb: 'D'` 副线门控 + `br: 'M'` 主线门控（Phase A 共通内容仅标 `sb: 'D'`）。
4. **神启专用「宗教」页签**：神启副线拥有独立的宗教页签（Religion tab），秘仪建筑、飞升门、秘典等内容集中在此页签展示，通过 `sb: 'D'` + `br: 'M'` 门控。研究仍走研究面板、配方仍走造物面板、灵术仍走灵术面板。

## 二、与主线的关系

### 核心原则：副线是加法

| 组合 | 效果 | 终局 |
|------|------|------|
| 灵修 + 秘仪 | 灵修循环 + 神恩加成 + 飞升阶梯（永久蜕变） | **神升**（替代灵迁） |
| 纯灵修 | 无神恩加成 | 灵迁 |

### 副线激活条件

- 研究「神启之学」：需政体已选 + 灵狐祠×3
- 激活后设 `G.subBranch.D = true`
- Phase B 自动进入秘仪路（因主线为灵修 `br: 'M'`）

### 净收益保证

- 神恩加成（+3%~80% 全局产出）远大于投入成本
- **秘仪路有主动代价**（第三门起有永久惩罚），但收益远大于代价
- 飞升阶梯的代价是密教模拟器的核心体验——**越接近飞升越要放弃凡俗**

## 三、神恩系统（共通核心机制）

### 定位

对标猫国太阳革命（Solar Revolution）：虔诚越多 → 全局产出加成越高。
Phase A 共通，两条路共享同一套神恩公式。

### 公式

```
graceBonus = min(graceCap, (sqrt(1 + piety / 100) - 1) × 50%)
```

- 默认上限 `graceCap = 50%`（升级可推至 80%）
- 需研究「恩典感召」后生效

| 虔诚存量 | 神恩加成 | 备注 |
|---------|---------|------|
| 0 | 0% | — |
| 25 | ~7% | Phase A 早期 |
| 100 | ~20% | Phase A 满配典型值 |
| 400 | ~36% | Phase B 中期 |
| 900 | ~50% | 默认上限 |
| 1600+ | ~64%→80% | 需升级提升上限 |

### 引擎集成

- 在 `calcR()` 中，全局乘数应用之后插入：
  ```js
  if (G.upg.graceLore?.done && G.res.piety) {
    var p = G.res.piety.v;
    var graceBonus = Math.min(G._graceCap || 0.50,
      (Math.sqrt(1 + p / 100) - 1) * 0.50);
    if (graceBonus > 0)
      for (const k of Object.keys(r)) if (r[k] > 0) r[k] *= (1 + graceBonus);
    G._graceBonus = graceBonus;
  }
  ```
- 仅作用于**正向产出**（`r[k] > 0`），不放大负向消耗

## 四、飞升阶梯（秘仪独占机制）

### 定位

五重门 = 五次不可逆的永久蜕变。每扇门需要大量秘知+神露，打开后永久改变角色状态。越接近飞升，代价越大。

核心决策：**什么时候攒够资源开门？是否愿意承受后期代价？**

### 五重门

| 阶 | 门名 | 消耗 | 永久效果 | 永久代价 | 风味 |
|----|------|------|---------|---------|------|
| 一 | 初窥之门 | 秘知×30, 神露×5 | 灵术效果 +10% | — | "你第一次听见了门后的声音" |
| 二 | 离形之门 | 秘知×80, 神露×15, 虔诚×100 | 全局产出 +8%, 研究速度 +15% | — | "你的影子开始独立行动" |
| 三 | 饮露之门 | 秘知×150, 神露×30 | 神恩上限 +15%, 解锁 Phase C 高阶配方 | 幸福上限 -5% | "你尝到了不属于凡间的味道" |
| 四 | 褪壳之门 | 秘知×300, 神露×60, 虔诚×300 | 灵修资源产出 ×1.5 | 人口上限 -5 | "你正在丢弃一些不再需要的东西" |
| 五 | 飞升之门 | 秘知×500, 神露×100, 虔诚×500 | 触发终局：**神升** | 人口上限 -10, 基础资源产出 -20% | "你不再是狐狸了" |

### 飞升规则

- 必须按顺序开门（一→二→三→四→五）
- 每扇门不可逆，开了就回不去
- 第三门起有永久代价——**飞升不是免费的**
- 第五门直接触发终局条件（但玩家仍可选择不立即触发终局事件）
- 化神池建筑等级 = 可开门的最高阶数（化神池×1 = 可开第一门，×5 = 可开第五门）

### 门与阶段的关系

```
Phase B：开第一、二门（无代价，纯正面）
Phase C：开第三门（首次代价），运用门后力量
Phase D：开第四、五门（重大代价），走向飞升
```

### 引擎集成

```js
// G._gates = 0  // 已开门数（0-5）
// G._gateEffects = { spellM, allM, researchM, graceCap, hapCap, maxPop, baseResM }
// 开门：检查资源+化神池等级 → 扣除资源 → G._gates++ → 写入 _gateEffects
// calcR() / calcH() / calcMx() 各自读取对应效果
```
