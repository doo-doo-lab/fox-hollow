### 开发规则

1. **不写短句（tip）**：除非明确要求，实现步骤时不主动编写 tip 短句/风味文字。
2. **遇到问题或抉择先查猫国百科**：设计不确定时优先参考 `reference/` 目录下的猫国建设者百科（猫咪百科、资源大全、常用资料等），作为机制设计的第一参照源。
3. **兼容游戏逻辑与资源体系**：所有新增内容必须与现有引擎机制（calcR/calcMx/calcEnergy/calcPollution 等）、资源生产消耗链、UI 渲染逻辑保持兼容，不引入孤立或断链的设计。

## 零、执行顺序

> **阅读说明：** 每个阶段内部严格按 `资源 → 研究 → 建筑 → 职业 → 配方 → 升级` 顺序实现。能量系统需先于所有 B 阶段建筑，污染系统可与 A 阶段首批建筑同步上线，E 阶段建筑必须跟随太空任务链逐个解锁，不可批量上线。

#### 步骤 11 Bug 修复记录

- **Bug#1（严重）：内燃机燃料耗尽时仍计入能量产出**。`calcEnergy()` 无条件累加所有建筑的 `energyP`，未检测燃料资源是否为 0。当火油耗尽后，内燃机变成免费能量源（+3/座），破坏能量平衡博弈。
  - **修复**：`calcEnergy()` 新增燃料门控——遍历建筑 `e` 中所有负向 `*P` 字段（排除 `energyP`/`pollutionP`），若对应资源 `v <= 0` 则该建筑不计入 `prod`。仅影响 `combustEngine`（需火油）和未来的 `熔炉`（需辉石）。`steamEngine`/`windTower` 无消耗型 `*P`，不受影响。
- **Bug#2（中等）：工厂配方加成不受能量衰减影响**。工厂 `energyC:2` 标记为耗能建筑，但 `craftAllM` 效果在 `calcR` 自动制作段和 `craft()` 手动制作中被直接读取，未乘以 `energyRatio`。缺能时工厂正向 `*P` 产出（无）确实被衰减，但实际核心功能（配方加成）逃逸了惩罚。
  - **修复**：`engine.js` calcR 自动制作段 + `engine-actions.js` craft() 中，`craftAllM` 加成在叠入 `craftMul`/`outMul` 前先乘以 `energyRatio`（当 < 1 时）。能量充足时无变化。

#### 步骤 12 Bug 修复记录

- **Bug#1（引擎不一致）：炉匠段工程师加成缺少 policyTrainFlat**。`engine.js` 炉匠自动炼钢段中，工程师 `eBonus` 计算为 `0.03 * c * (1 + train*0.1)` 而未加 `policyTrainFlat`；自动制作段和 `craft()` 手动制作中均包含了 `policyTrainFlat`。当玩家设置含 `trainFlat` 政策时，炉匠炼钢的工程师加成低于预期，且 UI 显示（含 `_ptf`）与引擎实际不一致。
  - **修复**：`engine.js` 炉匠段 `eBonus` 计算改为 `0.03 * c * (1 + train*0.1 + policyTrainFlat)`，与其他段一致。
- **Bug#2（UI 显示偏差）：炉匠 resBreakdown 缺少政体/政策职业乘数**。`ui.js` 炉匠显示段 `smRate` 公式为 `TPD/60 * smC * smTrain * happy * smSeason`，而引擎 `calcR` 中为 `TPD/60 * smC * smTrain * happy * polityJobMul`（再乘季节）。有政体时 UI 数字偏离实际。
  - **修复**：`ui.js` 炉匠段新增 `smPolityJobMul` 计算（镜像 calcR 政体/政策 jobM 逻辑），乘入 `smRate`。
- **设计确认**：机师不受染丝节令/祖灵影响（机械维护与季节无关）；工程师蓝本产出受全部乘数、配方加成只受训练+能量（有意区分，与猫国 Engineer 一致）。

## 三、能量系统

能量不是资源，是**产消平衡值**。每秒产出 - 每秒消耗 = 净能量。

- 净能量 ≥ 0：一切正常
- 净能量 < 0：所有吃能量的建筑产出按缺口比例衰减（猫国同机制）

## 四、污染机制（硬惩罚）

工业建筑每秒增加污染，治理建筑每秒减少污染。

### 惩罚阶梯

| 污染值 | 惩罚 |
|--------|------|
| 0-30 | 无 |
| 31-60 | 幸福度-10%, 野莓产出-15% |
| 61-100 | 幸福度-25%, 野莓产出-30%, 狐狸上限-2 |
| 101-150 | 幸福度-40%, 野莓产出-50%, 狐狸上限-5, 商队-20% |
| 150+ | 幸福度-60%, 野莓产出-70%, 狐狸上限-8, 研究-30%, 灰雾事件 |

### 烟囱机制

不减少污染，但提高"豁免阈值"。每座烟囱让惩罚阶梯整体右移 50 点。3 座烟囱 = 污染 150 以内无惩罚。治标不治本——污染持续涨，烟囱有上限。

### 灰雾事件（污染 150+）

随机触发，类似冬季熊袭击。可能导致：狐狸生病（-1 人口）、建筑损坏、资源损失。

### 核心博弈

- 多造工业建筑 → 产出爆炸 → 污染飙升 → 必须造治理建筑
- 治理建筑吃资源和建造位 → 挤占工业建筑空间
- 烟囱是短期方案，净化池/风力塔/净林是长期方案

### 工业线终局逻辑

工业线终局始终是**太空**，副线不改变终局，只提供额外叙事层和造价/前置调整：

| 副线组合 | 终局 | 叙事差异 |
|---------|------|---------|
| 无副线 | 太空 | 标准太空叙事 |
| +神启 | 太空 | 标准太空叙事（神启提供加法型加成，不改变终局叙事） |
| +通达 | 太空 | 太空叙事 + 通达叙事段落（万族送行、商路/盟约回顾） |
| +神启+通达 | 太空 | 太空叙事 + 通达叙事段落 + 神启叙事段落 |

这意味着：
- **H1-H6 在通达激活时提供额外前置/造价/叙事**，但不改变终局类型。
- 神启对工业线的影响是**加法型**（提供额外资源/加成/建筑），不是**改终局型**。
- 神启的具体加法效果（如祭祀加速生产、神迹提供稀有材料）由 branch-divine.md 定义，不影响本文档的 hook 结构。

### 覆写规则

1. **纯加法**：副线覆写优先"追加"而非"替换"。H1 追加前置条件，H5 追加升级。
2. **终局唯一**：工业线终局始终为太空，通达和神启只增加叙事段落和造价调整，不产生不同终局。
3. **不修改 A-D 内容**：副线 hook 全部在 E 阶段末期。A-D 阶段内容不受副线影响。
4. **资源命名空间**：副线引入的新资源（盟约典、神帆等）在各自副线文档中定义，不出现在本文档。
