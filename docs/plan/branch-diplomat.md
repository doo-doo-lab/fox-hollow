# 通达副线详细设计

> **副线定位**：纯辅助副线，不改变任何主线终局。通达是主线之上的外交叠加层，走通达的玩家在结局时会看到额外的通达叙事段落，但终局本身不变。
> 一句话：**你在主线之余，还编织了一张连接七族的邦交网——从偶尔来访的商队，到永驻共谷的盟友。**
> 通达体系在任意主线下运作一致，Phase A-D 不分流，工业和灵修玩家走同一套资源链、同一套建筑、同一套邦交系统。
> 体量约为主线的 50%。6 资源，14 建筑，11 配方，4 职业，27 研究，75 升级。

> ⚠️ **mulerun 接手开发须知**：本文档是**详细设计文档**，不是开发执行手册。
> 接手开发请先按顺序读：[INDEX.md](INDEX.md) → [roadmap-v2.md §四 实施进度表](roadmap-v2.md) → [DEV_SOP.md](DEV_SOP.md) → [RULES.md §八](../RULES.md)。
> 不要直接复制本文档代码到 data.js。通达副线实施在 Phase 3（v0.18）才开始，当前先聚焦 Phase 1。

### 开发规则

1. **不写短句（tip）**：除非明确要求，实现步骤时不主动编写 tip 短句/风味文字。
2. **遇到问题或抉择先查猫国百科**：设计不确定时优先参考 `reference/` 目录下的猫国建设者百科。
3. **兼容游戏逻辑与资源体系**：所有新增内容使用 `sb: 'T'` 副线门控。不再需要 `br: 'M'`/`br: 'I'` 主线门控，通达在任何主线下内容完全一致。
4. **外交建筑放「山外」页签**：外交建筑集中在山外页签（tab 'w'）展示。研究仍走研究面板、配方仍走造物面板。
5. **基于已有种族**：邦交对象为游戏中已存在的 7 个 NPC 种族（河獭·哗啦、白鹤·拂月、旧墟遗民、山猫·阿斑、雪鸮·夜眼、水鼠·囤囤、考拉·小曼），不另造新种族。

## 一、通达 vs 闭谷政策（外交域二选一）

参照猫国政策模型（外交 vs 孤立主义），外交域为二选一，各有正面加成，不存在纯负面。

| 选项 | 效果 | 适合 |
|------|------|------|
| **通达** | 解锁外交副线全部内容（声望系统、邦交系统、山外页签外交区），外交资源产出加成 | 愿意投入资源经营外交关系的玩家 |
| **闭谷** | 不解锁外交内容，但获得内政加成（基础资源产出 +5%，建筑花费 -3%） | 专注主线、不想分散精力的玩家 |

> 设计原则：两条路都是正面加成，不存在纯负面。闭谷不是惩罚，而是"把外交的资源省下来，换取内政效率"。

### 解锁条件

- 外交政策域在第五纪政策选择阶段出现
- 选择「通达」后设 `G.subBranch.T = true`
- 选择「闭谷」后设 `G.subBranch.T = false`，激活闭谷内政加成
- 与神启副线互不冲突（可同时开通达+神启）

## 二、与主线的关系

### 核心原则：副线是辅助加法，不改变终局

| 组合 | 效果 | 终局 |
|------|------|------|
| 工业 + 通达 | 工业产能 + 声望加成 + 邦交被动收益 | **太空**（不变，通达仅增加结局叙事） |
| 工业 + 通达 + 神启 | 同上 + 神恩加成 | **太空**（不变） |
| 工业 + 闭谷 | 工业产能 + 闭谷内政加成 | 太空 |
| 灵修 + 通达 | 灵修循环 + 声望加成 + 邦交被动收益 | **灵迁**（不变，通达仅增加结局叙事） |
| 灵修 + 通达 + 神启 | 三重叠加 | **神升**（不变，通达仅增加结局叙事） |
| 灵修 + 闭谷 | 灵修循环 + 闭谷内政加成 | 灵迁 |

### 通达叙事层

走了通达的玩家在进入主线终局时，会在结局画面中看到通达专属的额外段落：
- 工业线终局（太空）：七族送行、邦交功绩回顾
- 灵修线终局（灵迁/神升）：七族朝圣、盟约回响
- 不走通达则没有这些段落，但终局结局完全一致

### 主线无关

| 主线 | Phase A | Phase B | Phase C | Phase D |
|------|---------|---------|---------|---------|
| 工业 | 初交 | 结邦 | 深盟 | 共谷 |
| 灵修 | 初交 | 结邦 | 深盟 | 共谷 |

Phase A-D 完全一致，不根据主线分流。所有内容仅标 `sb: 'T'`。

## 四、声望系统（共通核心机制）

### 定位

对标猫国太阳革命（Solar Revolution）：声誉越多 → 外交产出加成越高。
Phase A-D 共享同一套声望公式。**无惩罚阶梯，无负面事件**。纯正面激励。

### 公式

```
reputeBonus = min(reputeCap, (sqrt(1 + renown / 100) - 1) × 40%)
```

- 默认上限 `reputeCap = 40%`（升级可推至 60%）
- 需研究「声望学」后生效
- 仅作用于通达副线资源的正向产出（不影响主线资源）

| 声誉存量 | 声望加成 | 备注 |
|---------|---------|------|
| 0 | 0% | — |
| 25 | ~6% | Phase A 早期 |
| 100 | ~16% | Phase A 满配 |
| 400 | ~29% | Phase B 中期 |
| 900 | ~40% | 默认上限 |
| 1600+ | ~51%→60% | 需升级提升上限 |

### 引擎集成

- 在 `calcR()` 中，全局乘数应用之后插入：
  ```js
  if (G.upg.reputeLore?.done && G.res.renown) {
    var rn = G.res.renown.v;
    var reputeBonus = Math.min(G._reputeCap || 0.40,
      (Math.sqrt(1 + rn / 100) - 1) * 0.40);
    if (reputeBonus > 0) {
      // 通达副线资源列表（统一，不分主线）
      var dipRes = ['credential','charter','exotic','allianceSeal','commonPact'];
      for (const k of dipRes) if (r[k] && r[k] > 0) r[k] *= (1 + reputeBonus);
    }
    G._reputeBonus = reputeBonus;
  }
  ```
- 仅作用于**正向产出**（`r[k] > 0`），不放大负向消耗

## 五、邦交系统（通达独占核心机制）

### 定位

将游戏已有的"商队来访→买卖→走人"浅层关系，深化为"长期邦交→互惠→共谷"。7 个已有种族就是邦交对象，现有商驿/驿道/远行是通达的前置基础设施。

核心决策：**集中资源深化少数几族，还是广铺浅层邦交？**

### 七族定义

基于游戏中已存在的 7 个 NPC 种族：

| # | 族名 | NPC名 | 已有交互 | 擅长领域 |
|---|------|-------|---------|---------|
| 1 | 河獭 | 哗啦 | 商队：卖丝帛/矿铁，收购木板 | 资源流通 |
| 2 | 白鹤 | 拂月 | 信使：卖卷轴/符咒，收购砖块 | 学识传递 |
| 3 | 旧墟遗民 | — | 商队：卖古币/符咒（不收购） | 古技传承 |
| 4 | 山猫 | 阿斑 | 竞拍：急性子抢手货 | 商贸竞争 |
| 5 | 雪鸮 | 夜眼 | 竞拍：神秘偏好罕物 | 暗夜秘知 |
| 6 | 水鼠 | 囤囤 | 竞拍：什么都要、易缩 | 囤积仓储 |
| 7 | 考拉 | 小曼 | 迟到段子（竞拍不参与） | 安逸悠闲 |

### 邦交深度（5 级）

| 深度 | 名称 | 消耗 | 效果 | 维护 |
|------|------|------|------|------|
| 1 | 结识 | 信物×5 + 声誉×30 | 该族商队来访率 +20%，解锁浅层加成 | 无 |
| 2 | 友邦 | 信物×12 + 邦书×3 + 声誉×80 | 浅层加成翻倍，该族商队售价 -10% | 无 |
| 3 | 盟邦 | 邦书×10 + 信物×20 + 声誉×150 | 解锁深层加成，该族专属远行目的地 | 邦书 .002/tick |
| 4 | 至交 | 邦书×25 + 盟印×3 + 声誉×300 | 深层加成加强，该族驻谷代表（永久 NPC） | 邦书 .005/tick |
| 5 | 永盟 | 盟印×8 + 邦书×40 + 声誉×600 | 深层加成极强，解锁终极被动 | 邦书 .010/tick |

### 邦交深化规则

1. 必须按深度顺序推进（1→2→3→4→5）
2. 深度 1-2：纯收益，无维护成本，低门槛
3. 深度 3-5：需每 tick 持续维护（邦书），断供时冻结效果（不撤销）
4. 深度 3 解锁该族专属远行目的地（7 个新目的地）
5. 深度 4 解锁驻谷代表（永久 NPC，远客居提供住所）
6. 同时可持有全部 7 族邦交，但深化资源极高
7. 实际上限约 2-3 族达到深度 5（资源约束，非硬性上限）

### 与已有系统挂钩

- **商队交易**：每次买/卖操作，对应族好感 +1（使馆存在时）
- **竞拍获胜**：对应族好感 +2
- **远行返回**：远行旧墟 → 旧墟遗民好感 +1；远行云岭 → 白鹤好感 +1
- **好感累积**：好感是深化的软性前置，好感 ≥ N 时才可深化到对应深度

## 六、通达资源链

```
现有经济基础
  商驿/驿道/瞭望塔（已有建筑）
  商队来访/竞拍（已有系统）
       ↓
A. 初交 ──→ 声誉（renown）、信物（credential）
       ↓
B. 结邦 ──→ 邦书（charter）+卷轴、异珍（exotic）+基础资源
       ↓
C. 深盟 ──→ 盟印（allianceSeal）+邦书+异珍
       ↓
D. 共谷 ──→ 共谷契（commonPact）+盟印+邦书+声誉
```

### 资源层级

```
声誉 → 信物 → 邦书(+卷轴) → 异珍(+基础资源)
                    ↓
              盟印(+邦书+异珍)
                    ↓
              共谷契(+盟印+邦书+声誉)
```

4 层嵌套（声誉→信物→邦书/异珍→盟印→共谷契），对标主线 5 层的 ~50%。

### 覆写规则

1. **纯加法**：通达覆写优先"追加"而非"替换"。追加前置、追加升级、追加叙事
2. **终局不变**：通达永远不改变终局类型（工业=太空，灵修=灵迁/神升）
3. **A-D 不受影响**：通达 hook 全部在终局阶段
4. **资源命名空间**：通达引入的替换资源在本文档定义
