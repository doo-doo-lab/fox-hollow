# v0.15 综合优化方案（修订版）

> 基于 v0.14.1 现状 + 路线图 v0.14→v0.26+ 总规划，统一解决以下问题：
> - 染丝/果酒/墨锭缺乏持续消耗
> - 香料获取来源过少
> - 丝帛获取瓶颈
> - 缺少提升商队来访概率/图纸出现概率的机制
> - 文化建筑线过于线性
> - 锻造炉/铁器冶炼循环依赖
> - 部分建筑解锁条件不合理
>
> 本方案定位：**路线图 v0.15 的完整实施内容**，与路线图原 v0.15"节气与节庆"合并。
> 节气系统（24 节气微事件）推迟至 v0.19 玄学阶段统一实施（与占卜/灵气流/节气玄解一体设计更合理）。

---

## 一、资源循环体系总览

改动后形成三环嵌套结构：

```
内环（基础循环）：野莓/圆木/碎石/兽皮 → 加工品 → 建筑/研究
中环（贸易循环）：香料/丝帛 ↔ 染丝/果酒/墨锭 → 节令消耗 + 灵术消耗 + 建筑维持
外环（灵性循环）：符咒 → 灵术 → 临时增益 → 推动内环/中环效率
```

核心设计原则（参考猫国）：
- 每种资源至少有 2 个消耗场景
- 稀有资源有 3 条以上获取路径
- 建筑有维持消耗，不是"建了就完事"
- 转化品消耗后产生增益，增益反过来加速基础生产（正反馈环）

## 二、香料获取扩展（5 条路径）

### 2.1 猎手「林间采风」（新机制）

每季结算时做一次概率判定：

```
发现概率 = min(0.5, 0.12 × 在岗猎手数 + 0.03 × 猎手授业等级)
成功 → 获得 1 香料（需 spice 上限 > 0 且未满）
```

| 猎手数 | 授业 0 | 授业 2 | 授业 4 |
|--------|--------|--------|--------|
| 1 | 12% | 18% | 24% |
| 2 | 24% | 30% | 36% |
| 3 | 36% | 42% | 48% |
| 4+ | 上限 50% | 50% | 50% |

- 解锁条件：猎手 ≥ 1 + 驿道 ≥ 1（即 spice.mx > 0）
- 期望产量：3 猎手 + 授业 1 级 ≈ 1.6 香料/年
- 主题：猎手在山林活动，发现野生香草

日志池：

```js
const HUNTER_SPICE_LOGS = [
  '猎手在溪谷背阴处发现了一丛野香草，小心翼翼地连根拔起。',
  '追猎物的路上踩到了什么，低头一看是长满了的香料苗。',
  '一只猎手说山坡上有股味道不对劲——是好的那种不对劲。',
];
```

### 2.2 商贩「路边生意」（新机制）

引擎追踪隐藏计数器 `G.merchantSpiceAcc`：

```
每 tick：G.merchantSpiceAcc += 在岗商贩数 × 实际铜钱产率（含授业/满意度/加成）
当 G.merchantSpiceAcc ≥ 15 时：
  G.merchantSpiceAcc -= 15
  35% 概率获得 1 香料（需 spice 上限 > 0 且未满）
```

- 解锁条件：商贩 ≥ 1 + 商驿 ≥ 1
- 期望产量：3 商贩（授业 1 级）≈ 0.6 香料/年
- 主题：商贩在做买卖过程中顺带从路过旅客手里换到香料

日志池：

```js
const MERCHANT_SPICE_LOGS = [
  '一个路过的旅狐用一小包香料抵了欠款，商贩欣然收下。',
  '集市角落换到了一撮来路不明的香料——闻着是正宗的。',
  '商贩说今天做了笔好买卖，顺手掏出一包带味儿的东西。',
];
```

### 2.3 商队到访赠礼（新机制）

任何商队到访时 40% 概率附赠 1 香料（需 spice.mx > 0）。

```js
// trySpawnCaravan() 生成商队成功后
if (G.res.spice.mx > 0 && Math.random() < 0.4) {
  G.caravan.gift = { r: 'spice', a: 1 };
}
```

日志附加："……顺手留下一小撮路上采的香草。"

- 期望产量：约 2 次商队/年 × 0.4 = 0.8 香料/年

### 2.4 旧墟远行掉落（数据改动）

`EXD.oldRuin.rewards` 新增：

```js
{ r: 'spice', min: 1, max: 1, prob: .25 }
```

主题依据：旧墟叙事碎片 #2 提到"铁针、针孔里留着炭化的线"——废墟中有手工业遗迹，残存香料合理。

- 期望产量：约 0.5 香料/年（取决于远行频率）

### 2.5 山猫行商购买（现有，不变）

2 香料 / 8 铜钱，每次商队驻留可购买 1 次。

### 香料供需平衡

| | 年产量 |
|---|--------|
| 猎手（3 人授业 1） | ~1.6 |
| 商贩（3 人商驿 1） | ~0.6 |
| 商队赠礼 | ~0.8 |
| 旧墟远行 | ~0.5 |
| 山猫购买 | ~1.0 |
| **总供给** | **~4.5** |

| | 年消耗 |
|---|--------|
| 节令系统（染丝→需香料） | 4 |
| 山谷宴席灵术 | 2-4 |
| 盈库灵术（染丝→需香料） | 2-4 |

供给 < 需求，玩家仍需主动购买山猫商队香料或用商风召商队来补缺。保持稀缺感。

## 三、丝帛获取扩展（3 条路径）

### 3.1 纺丝帛配方（新工坊配方）

```js
weave: {
  n: '纺丝帛', d: '兽皮+圆木 → 丝帛',
  inp: [{ r: 'leather', a: 8 }, { r: 'wood', a: 5 }],
  out: [{ r: 'silk', a: 1 }],
  uq: { b: { artistry: 1 } },
  tip: ['不是正经蚕丝，但搓出来一样滑溜。']

```

- 解锁条件：艺工坊 ×1（后期本地自给）
- 不构成循环依赖：解锁艺工坊之前通过河獭商队/互兑获取丝帛→做染丝→解锁百艺通觉→建艺工坊→解锁纺丝帛

### 3.2 香料互兑配方（新工坊配方）

```js
spiceToSilk: {
  n: '换丝帛', d: '香料+铜钱 → 丝帛',
  inp: [{ r: 'spice', a: 3 }, { r: 'coin', a: 5 }],
  out: [{ r: 'silk', a: 1 }],
  uq: { b: { tradePost: 1 } },
  tip: ['三袋香料加五枚钱，过路的河獭掏出一匹丝，双方满意。']

```

- 解锁条件：商驿 ×1（中期可用）
- 有损兑换：3 香料 + 5 铜钱 → 1 丝帛，防止套利
- 让铜钱参与循环，给铜钱增加消耗场景

### 3.3 河獭商队购买（现有，不变）

2 丝帛 / 10 铜钱。

### 3.4 远途跋涉研究费用下调

```js
// 原：p: [{ r: 'lore', a: 45 }, { r: 'silk', a: 2 }, { r: 'ancCoin', a: 1 }]
// 改：
```

## 四、节令系统（新机制）

### 4.1 机制

研究「百艺通觉」完成后自动启用。每季开始时，若对应文化资源存量 ≥ 1，自动扣除并给予本季加成：

| 消耗 | 加成 | 主题 |
|------|------|------|
| 染丝 ×1 | 全职业产出 +5% | 穿新衣干活有精神 |
| 果酒 ×1 | 野莓产量 +8% | 微醺的狐狸摘得更多 |
| 墨锭 ×1 | 学识产出 +10% | 研墨读书效率高 |
| 三者同时满足 | 额外满意度 +3% | 三全礼 |

- 有就扣，没有就跳过，部分满足给部分加成
- 年消耗：4 染丝 + 4 果酒 + 4 墨锭
- 分解到基础资源年消耗：4 丝帛 + 4 香料 + 120 野莓 + 12 兽皮 + 8 碎石

### 4.2 数据定义

```js
const SEASON_RITES = {
  dye:  { consume: 1, desc: '穿新衣干活有精神' },
  wine: { consume: 1, desc: '微醺的狐狸摘得更多' },
  ink:  { consume: 1, desc: '研墨读书效率高' },
};
```

### 4.3 引擎实现

季节切换时：

```js
G.seasonRites = {};
for (const [res, cfg] of Object.entries(SEASON_RITES)) {
  if (G.res[res].v >= cfg.consume) {
    G.res[res].v -= cfg.consume;
    G.seasonRites[res] = true;
  }
}
// 三全礼判定
G.seasonRites.all = G.seasonRites.dye && G.seasonRites.wine && G.seasonRites.ink;
```

`calcR()` 中应用：

```js
if (G.seasonRites.dye)  jobMul += 0.05;
if (G.seasonRites.wine) berryMul += 0.08;
if (G.seasonRites.ink)  loreMul += 0.10;
// calcH() 中
if (G.seasonRites.all) h += 0.03;
```

### 4.4 正反馈环

果酒加成野莓 +8% → 更多野莓 → 更多果酒原料 → 维持加成。
参考猫国"猫薄荷→精炼木→更多田→更多猫薄荷"的自我强化循环。

## 五、文化灵术（3 个全新效果维度）

### 设计原则

不与现有灵术效果重叠。现有灵术已覆盖：野莓产量(祈雨)、全职业产出(祖灵)、建筑造价(丰收祭)、远行时间(灵路)、商队召唤(商风)、满意度(山谷宴席)。新灵术覆盖三个空白维度：资源上限、工坊产出、研究费用。

### 5.1「盈库」

```js
overflow: {
  n: '盈库', d: '本季所有资源存储上限 +40%',
  cost: [{ r: 'dye', a: 2 }],
  uq: { u: { artistryLore: 1 } },
  tip: ['本来放不下了——但换个方式叠，居然又挤进去了。']
}
```

- 消耗：染丝 ×2
- 效果：本季所有资源 `mx` 临时乘以 1.4
- 每季限一次
- 使用时机：冬季前多囤野莓；建筑潮前让铜钱/砖块/木板一次攒够
- 无重叠：储藏窖/石窖是永久上限，月光井加产量不加上限

### 5.2「双工」

```js
doubleCraft: {
  n: '双工', d: '本季所有工坊制作产出 ×2',
  cost: [{ r: 'wine', a: 2 }],
  uq: { u: { artistryLore: 1 } },
  tip: ['一锤下去出两个，狐狸自己都吓了一跳。']
}
```

- 消耗：果酒 ×2
- 效果：本季手动和自动制作产出翻倍
- 每季限一次
- 使用时机：攒够大量原料后施放——原料越多收益越大
- 无重叠：祖灵加职业产出，不影响工坊配方；丰收祭减建筑造价，不影响工坊

### 5.3「墨契」

```js
inkPact: {
  n: '墨契', d: '下一次研究花费 -40%',
  cost: [{ r: 'ink', a: 2 }],
  uq: { u: { artistryLore: 1 } },
  tip: ['写一份契约跟学识讲价，居然成了。']
}
```

- 消耗：墨锭 ×2
- 效果：`G.inkPact = true`，下次成功研究时费用 ×0.6（乘法叠加学者「划重点」天赋），研究完成后 `G.inkPact = false`
- 不限每季，但施放后直到用掉前不可再施放
- 使用时机：只对最贵的研究用（同声相应 250 学识+30 卷轴 → -40% = 省 100 学识+12 卷轴）
- 无重叠：丰收祭减建筑造价；学者划重点是永久 -10%；墨契是一次性 -40%

### 灵术汇总

| 灵术 | 消耗 | 效果 | 空白维度 | 限制 |
|------|------|------|----------|------|
| 盈库 | 染丝×2 | 本季上限+40% | 资源上限 | 每季一次 |
| 双工 | 果酒×2 | 本季工坊×2 | 工坊产出 | 每季一次 |
| 墨契 | 墨锭×2 | 下次研究-40% | 研究费用 | 用掉前不可重复 |

三个灵术分别消耗三种不同文化资源，每个都强到玩家不舍得不用。

## 六、建筑维持消耗

参考猫国熔炉/反应堆的"建筑需要养"模式，为文化建筑引入维持成本。

### 6.1 月歌台

- 维持：1 果酒/年（每年第一个季节自动扣除）
- 正常效果：符咒 +0.003/s
- 断供效果：符咒产出归 0（建筑不消失，恢复供给后自动恢复）
- 主题：月夜歌咏需要酒

### 6.2 艺工坊

- 维持：1 染丝/季
- 正常效果：`craftCultureMul +0.05` × 座数
- 断供效果：`craftCultureMul = 0`（文化配方产出加成失效）
- 主题：作坊需要持续染料供应

### 实现方式

季节切换时检查并扣除，标记失效状态：

```js
// 月歌台：每年春季扣果酒
if (G.season === 0 && G.bld.moonStage.c > 0) {
  if (G.res.wine.v >= G.bld.moonStage.c) {
    G.res.wine.v -= G.bld.moonStage.c;
    G.moonStageActive = true;
  } else {
    G.moonStageActive = false;
  }
}

// 艺工坊：每季扣染丝
if (G.bld.artistry.c > 0) {
  if (G.res.dye.v >= G.bld.artistry.c) {
    G.res.dye.v -= G.bld.artistry.c;
    G.artistryActive = true;
  } else {
    G.artistryActive = false;
  }
}
```

`calcR()` 中：

```js
// 月歌台符咒产出
if (G.moonStageActive) r.charm += BD.moonStage.e.charmP * G.bld.moonStage.c * 5;
// else: 不计入

// 艺工坊加成
var craftCultureMul = G.artistryActive ? BD.artistry.e.craftCultureMul * G.bld.artistry.c : 0;
```

## 七、文化建筑线改为双分支

### 7.1 改动前（单链）

```
树荫堂 → 月令新酿 → 月歌台 → 岁时有常 → 百艺通觉 → 艺工坊
                                           → 俗成共庆 → 共聚堂
                                           → 连枝溯本 → 祖龛
```

### 7.2 改动后（双分支）

```
树荫堂
  ├─ 路线甲（歌与祭）
  │   月令新酿 → 月歌台
  │                └─ 连枝溯本 → 祖龛
  │
  └─ 路线乙（艺与聚）
      岁时有常 → 刻名碑
                   ├─ 百艺通觉 → 艺工坊
                   └─ 俗成共庆 → 共聚堂
```

### 7.3 具体解锁条件改动

| 研究/建筑 | 原条件 | 新条件 | 改动说明 |
|-----------|--------|--------|----------|
| 岁时有常 | `u:{calendar:1}, b:{moonStage:1}` | `u:{folkLore:1}, b:{storyTree:1}` | 与月令新酿平级，不再依赖月歌台 |
| 刻名碑 | `b:{storyTree:2}, u:{engraving:1}` | `b:{storyTree:1}, u:{engraving:1}` | 降低树荫堂数量门槛 |
| 连枝溯本 | `u:{engraving:1}, b:{shrine:3}` | `u:{calendar:1}, b:{shrine:3}` | 改为依赖月令新酿（路线甲） |
| 俗成共庆 | `u:{engraving:1}, custom:7` | `u:{engraving:1}, custom:5` | 降低风俗门槛（见下文分析） |

### 7.4 风俗门槛分析

改动后各路线可触达的风俗数：

- 树下初闻解锁 3 个：篝火夜歌、新衣节、共狩日
- 月令新酿解锁 2 个：春迁俗、谷雨宴
- 岁时有常解锁 5 个：铭石礼、月话课、守夜传统、老火传承、祖荫祭

走路线乙的玩家：树下初闻(3) + 岁时有常(5) = 8 个可激活风俗 → custom:5 轻松满足。
走路线甲的玩家：树下初闻(3) + 月令新酿(2) = 5 个可激活风俗 → custom:5 刚好满足。

两条路线都有合理路径解锁共聚堂。

### 7.5 设计收益

- 树荫堂之后面临第一个真正的战略选择：先灵性（符咒产出线）还是先世俗（满意度+工坊线）
- 两条路最终都要走完，但先后顺序影响中期节奏
- 与图纸系统的"不可撤销分岔"理念一致

## 八、锻造炉循环依赖修复

### 问题

锻造炉产出矿铁 0.008/s，铁器冶炼研究需要 10 矿铁。无铁匠时需等 21 分钟纯空转。

### 修复：首炉赠铁 + 研究降费

**8.1 首次建造赠铁**

锻造炉从 0→1 时赠送 3 矿铁：

```js
// build() 中
if (id === 'smithy' && G.bld.smithy.c === 1) {
  G.res.iron.v = Math.min(G.res.iron.v + 3, G.res.iron.mx);
  log('锻造炉第一炉出铁了，获得 3 矿铁。');
}
```

- 3 铁可立即用于建造灵狐祠（需 3 铁），获得即时正反馈
- 主题："第一炉出铁"是冶金里程碑

**8.2 铁器冶炼研究降费**

```js
// 原：p: [{ r: 'lore', a: 40 }, { r: 'iron', a: 10 }]
// 改：
p: [{ r: 'lore', a: 40 }, { r: 'iron', a: 5 }]
```

改后流程：建炉送 3 铁 → 再攒 2 铁（约 1-2 分钟）→ 完成研究。空转期从 21 分钟缩短到 1-2 分钟。

## 十二、商队来访概率与图纸出现概率加成

### 12.1 现状分析

当前游戏中没有任何机制可以提升：
- 商队来访概率（固定每季 15% 判定）
- 图纸/专精出现概率（固定远行返回 20% 判定 + 特定建筑数量阈值）

这两项是中后期玩家最渴望提升的"元概率"，因为它们直接影响游戏节奏。

### 12.2 商队来访概率加成

**方案：挂载在「共聚堂」建筑上**

理由：共聚堂主题是"狐狸聚集之地"，外来商队也更愿意造访热闹的村庄。

```js
// BD.communityHall 新增效果
communityHall: {
  // 现有：满意度 +3%/座
  // 新增：
  e: { happyP: .03, caravanProb: .02 },
  // 每座共聚堂 +2% 商队来访概率（加法叠加，上限 +10%）
}
```

引擎实现（`engine-systems.js` trySpawnCaravan）：

```js
const baseProb = 0.15;
const bonus = Math.min(0.10, (G.bld.communityHall?.c || 0) * 0.02);
if (Math.random() < baseProb + bonus) { /* 生成商队 */ }
```

- 3 座共聚堂：15% → 21%（+40% 相对提升，体感明显）
- 5 座上限：15% → 25%（不会破坏稀缺感）
- 对应年均商队次数：2.4 → 4.0 次/年

**备选位置**（若觉得共聚堂太晚）：商驿也可分摊部分加成（每座 +1%）。但商驿已有铜钱产出效果，不宜再加过多。

### 12.3 图纸出现概率加成

**方案：挂载在「刻名碑」建筑效果 + 新增「墨契」灵术的隐藏加成**

理由：刻名碑主题是"铭记与传承"，记录得越多越容易发现图纸中的灵感；墨契灵术主题是"与知识讨价还价"。

```js
// BD.monument 追加效果
monument: {
  // 现有：全村产出 +(已激活习俗 ×0.2%)/座
  // 新增：
  e: { customAllM: .002, blueprintProb: .03 },
  // 每座刻名碑 +3% 图纸发现概率（加法叠加，上限 +15%）
}
```

引擎实现（远行返回图纸判定处）：

```js
const baseBpProb = 0.20;
const bpBonus = Math.min(0.15, (G.bld.monument?.c || 0) * 0.03);
if (Math.random() < baseBpProb + bpBonus) { /* 发现图纸 */ }
```

- 3 座刻名碑：20% → 29%
- 5 座上限：20% → 35%（显著但不过分）

**「墨契」灵术额外效果**：施放墨契后，下次远行返回时图纸概率额外 +20%（一次性，与建筑加成加法叠加）。

```js
// 墨契施放时
G.inkPact = true;
G.inkPactBp = true; // 额外标记

// 远行返回时
const bpBonusSpell = G.inkPactBp ? 0.20 : 0;
if (Math.random() < baseBpProb + bpBonus + bpBonusSpell) { /* 发现图纸 */ }
G.inkPactBp = false; // 用掉
```

这让墨契变成"多用途"灵术（研究-40% + 图纸概率），玩家需在"省研究费"和"提图纸率"之间权衡使用时机。

### 12.4 汇总

| 加成维度 | 来源 | 数值 | 上限 |
|----------|------|------|------|
| 商队来访概率 | 共聚堂 ×座数 | +2%/座 | +10% |
| 图纸出现概率 | 刻名碑 ×座数 | +3%/座 | +15% |
| 图纸出现概率 | 墨契灵术 | +20%（一次性） | — |

## 十四、前端展示策略（玩家认知负担管理）

### 14.1 设计原则

增量式揭露（Progressive Disclosure）：玩家只在需要时看到新机制，不一次性堆砌。

### 14.2 各机制的 UI 呈现

| 机制 | 展示方式 | 何时可见 |
|------|----------|----------|
| 猎手林间采风 | 仅日志一行文字（"猎手在溪谷发现野香草…+1 香料"） | 触发时 |
| 商贩路边生意 | 仅日志一行文字 | 触发时 |
| 节令系统 | 季节切换时顶部 toast 提示："穿新衣干活有精神 ▸ 全职业 +5%" | 季节切换时 |
| 节令加成状态 | 资源面板旁小图标（染丝=衣/果酒=杯/墨锭=笔），hover 显示效果 | 季节内持续 |
| 文化灵术 | 与现有灵术完全同一 UI 位置（营火 Tab → 灵术区），无额外入口 | 解锁后 |
| 建筑维持 | 建筑卡片右上角状态标签："运作中" / "❌ 断供"（红色） | 建造后 |
| 维持断供提示 | 日志警告 + 建筑列表该建筑名字变灰 | 断供时 |
| 商队概率加成 | 共聚堂 tooltip 中写明"+2% 商队来访"，不额外弹窗 | 建造后 |
| 图纸概率加成 | 刻名碑 tooltip 中写明"+3% 图纸发现"，不额外弹窗 | 建造后 |
| 双分支路线 | 研究面板中两条路线自然并列（现有 UI 已支持），无额外引导 | 解锁时 |

### 14.3 避免"机制过于复杂"的策略

1. **零新页签**：所有新内容嵌入现有 UI 位置（灵术区/工坊区/建筑列表/日志），不新增顶部 Tab
2. **被动触发为主**：猎手采风/商贩路边生意/节令/建筑维持全部自动结算，玩家无需手动操作
3. **日志即教学**：每个新机制首次触发时，日志用完整句子解释发生了什么（之后精简）
4. **tooltip 深度**：所有数值效果只在 hover 时展示详细公式，表面只显示结果
5. **渐进解锁**：
   - 玩家建了艺工坊 → 才看到纺丝帛配方
   - 玩家完成百艺通觉 → 才看到 3 个文化灵术
   - 玩家建了共聚堂 → 才在 tooltip 看到商队概率加成
