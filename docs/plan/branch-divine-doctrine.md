# 教团路详细设计（工业+神启副线）

> 副线定位：工业主线之上的信仰叠加层。信仰形态 = 组织化宗教狂热驱动建造。
> 对标：中世纪天主教廷大教堂运动 × Warhammer 40K 机械神教。
> 一句话：**你不是在经营工厂，你是在为神建造圣城。**
> 4 阶段递进，体量约为主线的 70%。
> 8 资源，22 建筑，15 配方，6 职业，35 研究，100 升级。

> ⚠️ **mulerun 接手开发须知**：本文档是**详细设计文档**，不是开发执行手册。
> 接手开发请先按顺序读：[INDEX.md](INDEX.md) → [roadmap-v2.md §四 实施进度表](roadmap-v2.md) → [DEV_SOP.md](DEV_SOP.md) → [RULES.md §八](../RULES.md)。
> 不要直接复制本文档代码到 data.js。教团路实施在 Phase 4（v0.19）才开始，当前先聚焦 Phase 1。

### 开发规则

1. **不写短句（tip）**：除非明确要求，实现步骤时不主动编写 tip 短句/风味文字。
2. **遇到问题或抉择先查猫国百科**：设计不确定时优先参考 `reference/` 目录下的猫国建设者百科。

4. **神启专用「宗教」页签**：神启副线拥有独立的宗教页签（Religion tab），教团建筑、教令、圣物等内容集中在此页签展示，通过 `sb: 'D'` + `br: 'I'` 门控。研究仍走研究面板、配方仍走造物面板。

## 二、与主线的关系

### 核心原则：副线是加法

| 组合 | 效果 | 终局 |
|------|------|------|
| 工业 + 教团 | 工业产能 + 神恩全局加成 + 教令系统 + 圣战远征 | 太空 + 神佑加成（不改终局类型） |
| 纯工业 | 无神恩加成 | 太空 |

### 副线激活条件

- 研究「神启之学」：需政体已选 + 灵狐祠×3
- 激活后设 `G.subBranch.D = true`
- Phase B 自动进入教团路（因主线为工业 `br: 'I'`）

### 净收益保证

- 神恩加成（+3%~80% 全局产出）远大于投入成本
- 教令系统为纯正面临时buff，无负面事件
- 教团路无惩罚，纯机会成本（资源分散+管理复杂度）

## 三、神恩系统（共通核心机制）

### 定位

对标猫国太阳革命（Solar Revolution）：虔诚越多 → 全局产出加成越高。
**无惩罚阶梯，无负面事件**。纯正面激励。

### 公式

```
graceBonus = min(graceCap, (sqrt(1 + piety / 100) - 1) × 50%)
```

- 默认上限 `graceCap = 50%`（升级可推至 80%）
- 需研究「恩典感召」后生效

| 虔诚存量 | 神恩加成 | 备注 |
|---------|---------|------|
| 0 | 0% | — |
| 25 | ~7% | Phase A 早期 |
| 100 | ~20% | Phase A 满配典型值 |
| 400 | ~36% | Phase B 中期 |
| 900 | ~50% | 默认上限 |
| 1600+ | ~64%→80% | 需升级提升上限 |

### 引擎集成

- 在 `calcR()` 中，全局乘数应用之后插入：
  ```js
  if (G.upg.graceLore?.done && G.res.piety) {
    var p = G.res.piety.v;
    var graceBonus = Math.min(G._graceCap || 0.50,
      (Math.sqrt(1 + p / 100) - 1) * 0.50);
    if (graceBonus > 0)
      for (const k of Object.keys(r)) if (r[k] > 0) r[k] *= (1 + graceBonus);
    G._graceBonus = graceBonus;
  }
  ```
- `G._graceCap` 默认 0.50，升级可修改
- 仅作用于**正向产出**（`r[k] > 0`），不放大负向消耗

### 与主线系统的隔离

- 不影响能量计算（神恩乘数在 calcEnergy 之后）
- 不影响污染计算（独立系统）
- 仅作用于资源产出速率

## 四、教令系统（教团独占机制）

### 定位

教令 = 消耗虔诚+圣火 → 发布一条限时全局buff。类比猫国「赞美太阳」但更细分：多条教令可选，同时只能激活有限条数（默认 1 条，教令堂扩展）。

核心决策：**当前局面发哪条教令最划算？**

### 教令规则

- 同时激活上限：默认 1 条，教令堂每座 +1（上限 4 条）
- 同一教令不可叠加，但不同教令可共存
- 冷却：每条教令发布后有 2 季冷却（不影响其他教令）
- **无负面事件**，纯机会成本

## 七、Phase A 详细设计（共通）

### 步骤 1：神恩系统

见第三节。实现要点：
- `G._graceBonus`：当前神恩加成值（缓存，每 tick 更新）
- `G._graceCap`：神恩上限（默认 0.50，升级可提升）
- `resetG()`：初始化 `G._graceBonus = 0; G._graceCap = 0.50;`
- `migrate()`：兜底 `G._graceBonus = G._graceBonus || 0; G._graceCap = G._graceCap ?? 0.50;`
- UI：虔诚资源 tooltip 中显示当前神恩加成百分比
